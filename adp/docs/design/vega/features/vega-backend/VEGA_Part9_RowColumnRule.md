# 二、需求分析

## 2.1 需求背景

数据视图需要对接各上层业务，实现对如 AF
视图业务功能（授权管理）的支持。平台本身也需要具备数据的访问控制能力，保证数据安全。

## 2.2 设计目标

1.  功能实现：设计一套标准化接口，覆盖AF等上层业务对于组织和人数据权限的使用。​

2.  功能优化：对 AnyFabric
    现有授权功能进行梳理，识别并剥离不合理的功能，精简设计复杂度，提升可维护性。​

## 2.3 需求价值

1.  降低维护成本：通过剥离冗余功能，简化系统架构，减少潜在的维护问题和故障点，提升系统稳定性。​

2.  数据安全管控：构建完善的数据访问控制能力，保障核心数据资产安全。

## 2.4 业务目标

1.  实现对视图行列级权限的支持。

2.  基于isf框架建立统一的用户和部门操作鉴权体系。

# 四、设计方案

## 4.1 AF 业务逻辑梳理

1.  授权和授权仅分配

    1.  授权：         
         能够授权给别人行列规则、新建、删除、查看行列规则、编辑已有的行列规则

    2.  授权仅分配：能够授权给别人行列规则、新建、删除、查看行列规则、不能编辑已有的行列规则

2.  创建行列规则的时候授权范围可以选择视图或者已有的行/列规则（选择已有行列规则也不是继承关系，是相当于按照一个模板创建）。

## 4.2 业务逻辑设计

1.  首先重新定义授权仅分配，其次这俩业务概念在 DIP
    是不同表达，参考下图对照。

    1.  授权：         
         能够授权给别人行列规则、查看行列规则、新建、编辑、删除已有的行列规则。

    2.  授权仅分配：能够授权给别人行列规则、查看行列规则、不能新建、编辑、删除已有的行列规则

2.  创建行列规则的时候授权范围只能选择视图，可在前端提供复制功能一键复制其他的行列规则模板。

  ---------------------------------------------------------
  **对照表**   **AF定义**   **DIP定义**
  ------------ ------------ -------------------------------
  1            授权         视图行列规则管理+行列规则授权

  2            授权进分配   视图行列规则授权
  ---------------------------------------------------------

- 视图权限点，红色为新增：新建、编辑、删除、查看、权限管理、数据查询、导入、导出、行列规则管理、行列规则授权。

### 4.3.2 权限码

数据视图权限码，标红加粗为新增。

  -------------------------------------------------------------------------------------------------------------------------------
       **操作**       **编码（唯一标识）**   **说明**
  ---- -------------- ---------------------- ------------------------------------------------------------------------------------
  1    查看           view_detail            获取资源的信息、具体的配置等

  2    新建           create                 生成一个新的资源实例，如：上传文档、新建资源等。

  3    编辑           modify                 更新已有资源的内容或属性，如修改文件、调整参数等。

  4    删除           delete                 删除已存在的资源。

  5    数据查询       data_query             指标模型、链路模型等数据资源对象的数据内容查询

  6    权限管理       authorize              赋予用户或角色对资源的操作权限，如分配文件访问权限。有"权限配置"权限的，必须此字段

  7    导入           import                 

  8    导出           export                 

  9    行列规则管理   rule_manage            对视图行列规则增删改查

  10   行列规则授权   rule_authorize         赋予用户或角色对视图行列规则的操作权限

USE dip_mdl;

\-- 视图行列规则表

CREATE TABLE IF NOT EXISTS t_data_view_row_column_rule (

f_rule_id varchar(40) NOT NULL DEFAULT \'\' COMMENT \'视图行列规则 id\',

f_rule_name varchar(255) NOT NULL COMMENT \'视图行列规则名称\',

f_view_id varchar(40) NOT NULL COMMENT \'视图 id\',

f_tags varchar(255) NOT NULL DEFAULT \'\' COMMENT \'标签\',

f_comment varchar(255) NOT NULL DEFAULT \'\' COMMENT \'备注\',

f_fields longtext NOT NULL COMMENT \'列\',

f_row_filters text NOT NULL COMMENT \'行过滤规则\',

f_create_time bigint(20) NOT NULL DEFAULT 0 COMMENT \'创建时间\',

f_update_time bigint(20) NOT NULL DEFAULT 0 COMMENT \'更新时间\',

f_creator char(36) NOT NULL COMMENT \'创建人id\',

PRIMARY KEY (f_rule_id),

UNIQUE KEY uk_f_rule_name (f_rule_name, f_view_id)

) ENGINE = InnoDB DEFAULT CHARSET = utf8mb4 COLLATE = utf8mb4_bin
COMMENT = \'数据视图行列规则\';

## 9.2 数据库升级

### 9.2.1 表结构

新增表 t_data_view_row_column_rule

### 9.2.2 数据升级

1.  资源类型为 data_view 的对象，要新增权限操作
    rule_manage、rule_authorize。

2.  新增资源类型 data_view_row_column_rule 。

3.  数据平台业务角色数据管理员、~~应用管理员~~的默认权限操作新增视图行列规则管理（rule_manage）、行列规则授权（rule_authorize ）。

4.  **已有的视图资源实例，对于其中有授权权限的视图，给其加上行列规则管理和行列规则授权的权限。**
