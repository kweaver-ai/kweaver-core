# 执行命令与环境变量说明

本文说明如何在项目根目录（`at-framework-0326`）安装依赖、配置环境、运行用例及生成报告。适用于 **YAML 用例**（如 `testcase/etrino`）与 **pytest + test_run.py** 入口。

---

## 1. 前置条件

- **Python 3.x**
- 安装依赖（在项目根目录执行）：

```bash
pip install -r requirement.txt
```

- **Allure**（仅在使用 `python main.py` 生成 HTML 报告时需要本机已安装 `allure` 命令）。

---

## 2. 配置文件

- 主配置：`config/config.ini`
- 静态 Token 解析优先级（与 `common/at_env.py` 一致）：
  1. 环境变量 `AT_API_TOKEN` 或 `API_ACCESS_TOKEN`
  2. `[external]` → `token`
  3. `[test_data]` → `application_token`

请求 Host / 协议：可被 `AT_SERVER_HOST`、`AT_REQUEST_SCHEME` 覆盖，否则读取 `[env]` 或 `[server]`。

---

## 3. pytest 直接执行

**工作目录**：项目根目录（含 `test_run.py`、`conftest.py` 的目录）。

### 3.1 运行当前默认模块全部用例

`[env].case_file` 或环境变量 `CASE_FILE` / `AT_CASE_FILE` 指向模块目录时：

```bash
python -m pytest test_run.py -s
```

### 3.2 指定模块目录

```bash
python -m pytest test_run.py -s --case-file "./testcase/etrino"
```

### 3.3 只跑某一个套件文件（推荐）

套件筛选使用 YAML 中的 **`story`**，或与 **`suites` 下文件名（不含 `.yaml`）** 一致，例如 `新增数据源.yaml`：

```bash
python -m pytest test_run.py -s --case-file "./testcase/etrino" --suite "新增数据源"
```

静默模式（少输出）：

```bash
python -m pytest test_run.py -q --case-file "./testcase/etrino" --suite "新增数据源"
```

### 3.4 只跑若干条用例（按 name）

`name` 必须与 YAML 中 **完全一致**（逗号分隔，无多余空格）：

```bash
python -m pytest test_run.py -s --case-file "./testcase/etrino" --case-names "用例名一,用例名二"
```

### 3.5 其它筛选（环境变量）

与命令行二选一或叠加时，以 `conftest` 中约定为准（pytest 参数会写回环境变量）：

| 变量 | 含义 |
|------|------|
| `CASE_FILE` / `AT_CASE_FILE` | 模块目录 |
| `SUITE` | 同 `--suite` |
| `CASE_NAMES` | 逗号分隔用例名 |
| `SCOPE` | scope id |
| `TAGS` | 逗号分隔标签 |
| `API_NAME` | `apis.yaml` 中的接口 `name` |
| `API_PATH` | URL 路径子串 |

示例（PowerShell）：

```powershell
$env:CASE_FILE = "./testcase/etrino"
$env:SUITE = "新增数据源"
python -m pytest test_run.py -s
```

---

## 4. 入口脚本（pytest + Allure）

```bash
python main.py --case-file "./testcase/etrino" --suite "新增数据源"
```

可选参数与 `pytest` 侧一致：`--scope`、`--tags`、`--api-name`、`--api-path`、`--case-names`、`--suite`。

报告目录由 `common/constant.py` 约定（通常为 `report/xml`、`report/html`）。

---

## 5. 环境变量统一配置（`common/at_env.py`）

**同一键：环境变量优先于 `config.ini` 对应项。**

| 变量 | 说明 |
|------|------|
| `AT_API_TOKEN` / `API_ACCESS_TOKEN` | 静态 API Token（不要带 `Bearer ` 前缀；若误带会自动去掉） |
| `AT_AUTH_SOURCE` / `AUTH_TOKEN_SOURCE` | `login`：默认先 `get_token`；`static`：仅用静态 Token |
| `AT_ADMIN_USER` / `AT_ADMIN_PASSWORD` | `get_token` 使用的账号密码 |
| `AT_SERVER_HOST` / `SERVER_HOST` | 被测 API Host（及 OAuth 登录侧解析） |
| `AT_REQUEST_SCHEME` | `https` 或 `http` |
| `AT_CASE_FILE` | 与 `CASE_FILE` 相同含义 |
| `AT_CLEAN_UP` | `1` 开启会话级清理 |
| `AT_CLEAN_UP_MODULE` | 仅当模块名匹配时清理 |

**PowerShell 示例**（静态 Token + 指定套件）：

```powershell
cd D:\Code\dip\code\kweaver\at-framework-0326
$env:AT_SERVER_HOST = "10.4.111.124"
$env:AT_API_TOKEN = "你的access_token"
$env:AT_AUTH_SOURCE = "static"
python -m pytest test_run.py -s --case-file "./testcase/etrino" --suite "新增数据源"
```

**套件级 `token_source: login`**（见套件 YAML）：每条用例会再调用 `get_token`，失败时回退到与默认 Bearer 相同的静态解析链（`AT_API_TOKEN` → `external.token` → `application_token`）。

```bash
python -m pytest tests/
```

---

## 7. 常见问题

- **`Authorization: Bearer ` 后面为空**：未设置 `AT_API_TOKEN` / `API_ACCESS_TOKEN`，且 `[external].token` 与 `[test_data].application_token` 均为空；或 `get_token` 失败且静态链仍无值。
- **`unrecognized arguments: --case-file`**：`conftest.py` 需包含 `pytest_addoption` 注册；请使用本仓库根目录的 `conftest.py` 执行。
- **套件 0 条用例**：`--case-file` 指向的目录不存在，或 YAML 中 `switch: n`，或接口名不在 `apis.yaml`。

更完整的架构与智能体筛选说明见同目录下《测试框架架构_全自动与智能体规范》等文档。
