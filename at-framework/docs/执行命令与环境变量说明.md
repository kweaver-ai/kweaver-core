# 执行命令与环境变量说明

本文说明在本仓库**项目根目录**（含 `test_run.py`、`conftest.py`、`main.py`）如何安装依赖、配置、运行 YAML 用例与生成报告。入口为 **pytest + `test_run.py`**；`main.py` 在 pytest 之外额外调用 Allure 生成 HTML。

---

## 1. 前置条件

| 项 | 说明 |
|----|------|
| Python | 3.x |
| 依赖 | 根目录执行：`pip install -r requirement.txt` |
| Allure | 仅在使用 `python main.py` 生成 HTML 报告时需本机已安装 `allure` 命令 |

---

## 2. 配置文件 `config/config.ini`

当前框架**不依赖** `[env]` 段；常用段落如下。

| 段落 / 键 | 作用 |
|-----------|------|
| `[server].host` | 被测 API 主机（不含协议）。可被 `AT_SERVER_HOST` / `SERVER_HOST` 覆盖。 |
| `[server].base_url` | 完整基址 URL。未设置 `AT_REQUEST_SCHEME` 时，**协议**由此 URL 推导（如 `https://10.0.0.1` → `https`）。 |
| `[test_data].application_token` | 静态 Access Token（不含 `Bearer `）。可被 `API_ACCESS_TOKEN` 覆盖。 |
| `[test_data].admin_user` / `admin_password` | `get_token` 缺省账号；可被 `AT_ADMIN_USER` / `AT_ADMIN_PASSWORD` 覆盖。 |

其它段落（如 `[database]`、`[log]`）按模块需要自行维护，与本文「请求 / 鉴权」无关的不展开。

---

## 3. 优先级总览

执行时建议记住三条规则：

1. **pytest 命令行参数** 优先于 **环境变量** 优先于 **默认值**（见 `conftest.py` 的 `pytest_configure`）。
2. **静态 Token**：`API_ACCESS_TOKEN` > `[test_data].application_token`（见 `common/at_env.py` 的 `static_access_token`）。
3. **默认鉴权方式**：`AT_AUTH_SOURCE` / `AUTH_TOKEN_SOURCE` 未设置时视为 **`static`**（仅用静态 Token）；设为 `login`（或兼容别名，见下文套件级）时再走 `get_token`。

---

## 4. pytest 直接执行

**工作目录**：项目根目录。

### 4.1 默认模块目录

未传 `--case-file` 时，模块目录来自：

`AT_CASE_FILE` → `CASE_FILE` → 默认 **`./testcase/etrino`**。

```bash
python -m pytest test_run.py -s
```

### 4.2 指定模块与其它参数

```bash
python -m pytest test_run.py -s --case-file "./testcase/etrino"
python -m pytest test_run.py -s --case-file "./testcase/etrino" --suite "新增数据源"
python -m pytest test_run.py -q --case-file "./testcase/etrino" --suite "新增数据源"
python -m pytest test_run.py -s --case-file "./testcase/etrino" --case-names "用例名一,用例名二"
```

**套件筛选**：与 YAML 中 **`story`** 一致，或与 `suites/` 下**文件名（不含 `.yaml`）**一致。

### 4.3 pytest 注册的选项（与 `conftest.py` 一致）

| 选项 | 说明 |
|------|------|
| `--case-file` | 模块用例目录，如 `./testcase/vega` |
| `--suite` | 套件 story 或 suites 下文件名（不含 `.yaml`） |
| `--case-names` | 逗号分隔用例 `name`，须与 YAML 完全一致 |
| `--scope` | scope id |
| `--tags` | 逗号分隔标签 |
| `--api-name` | `apis.yaml` 中的接口 `name` |
| `--api-path` | URL 路径子串 |

传入上述选项时，conftest 会把对应值**写回环境变量**（如 `SUITE`、`CASE_NAMES`），供 `compute_case_list` 使用。

### 4.4 仅用环境变量筛选（与命令行二选一或配合）

| 变量 | 含义 |
|------|------|
| `CASE_FILE` / `AT_CASE_FILE` | 模块目录 |
| `SUITE` | 同 `--suite` |
| `CASE_NAMES` | 逗号分隔用例名 |
| `SCOPE` | scope id |
| `TAGS` | 逗号分隔标签 |
| `API_NAME` | 接口 `name` |
| `API_PATH` | URL 路径子串 |

在运行 pytest 前于终端中设置上述变量即可（POSIX：`export VAR=value`）。

---

## 5. 入口脚本 `main.py`（pytest + Allure）

```bash
python main.py --case-file "./testcase/etrino" --suite "新增数据源"
```

参数与 pytest 侧一致：`--scope`、`--tags`、`--api-name`、`--api-path`、`--case-names`、`--suite`。

报告路径由 `common/constant.py` 约定（一般为 `report/xml`、`report/html`）；`main.py` 会先跑 pytest 再执行 `allure generate`。

---

## 6. 环境变量说明（`common/at_env.py`）

下列变量为框架**显式读取**项；未列出的键仍以 `config.ini` 为准（如数据库等）。

### 6.1 鉴权与 Token

| 变量 | 说明 |
|------|------|
| `API_ACCESS_TOKEN` | 静态 Token；不要带 `Bearer ` 前缀（若误带会自动去掉） |
| `AT_AUTH_SOURCE` / `AUTH_TOKEN_SOURCE` | `login`：会话默认 Bearer 先尝试 `get_token`；`static` 或**未设置**：仅用静态 Token |
| `AT_ADMIN_USER` / `AT_ADMIN_PASSWORD` | 覆盖 `[test_data]` 中账号密码，供 `get_token` 使用 |

**套件级 `token_source: login`**（套件 YAML）：单条用例会再调 `get_token`；失败时回退为与默认 Bearer 相同的静态链：`API_ACCESS_TOKEN` → `[test_data].application_token`。

### 6.2 请求目标（Host / 协议）

| 变量 | 说明 |
|------|------|
| `AT_SERVER_HOST` / `SERVER_HOST` | 覆盖 `[server].host` |
| `AT_REQUEST_SCHEME` | `https` 或 `http`；未设置时从 `[server].base_url` 推导，再不行则为 `https` |

### 6.3 模块路径与筛选

| 变量 | 说明 |
|------|------|
| `AT_CASE_FILE` | 与 `CASE_FILE` 同义，任设其一即可 |

（筛选类变量见上文 4.4。）

### 6.4 会话清理

| 变量 | 说明 |
|------|------|
| `AT_CLEAN_UP` | 设为 `1` 且本会话实际跑了 `test_run` 用例时，尝试调用模块下 `framework_hooks.session_clean_up` |
| `AT_CLEAN_UP_MODULE` | 若设置，仅当**当前模块目录名**与其一致时才执行清理；未设置则不对模块名做限制 |

### 6.5 加载行为（可选）

| 变量 | 说明 |
|------|------|
| `AT_STRICT_LOAD_APIS` | 设为 `1` 或 `true` 时，用例引用的接口名不在 `apis.yaml` 中会**加载失败**（CI 常用）；默认多为告警或跳过 |

---

## 7. 框架自测（不加载业务 YAML）

```bash
python -m pytest tests/
```

仅跑 `tests/` 下单元测试；不会按业务模块加载 YAML，也不会触发 `test_run` 相关的会话清理逻辑。

---

## 8. 常见问题

| 现象 | 可能原因 |
|------|----------|
| `Authorization: Bearer ` 后为空 | 未设 `API_ACCESS_TOKEN` 且 `[test_data].application_token` 为空；或 `AT_AUTH_SOURCE=login` 但 `get_token` 失败且静态链仍无值 |
| `unrecognized arguments: --case-file` | 未在项目根目录执行，或使用的 `conftest.py` 不是本仓库根目录版本 |
| 套件 0 条用例 | `--case-file` 目录不存在；YAML `switch: n`；接口 `name` 不在 `apis.yaml`；筛选条件过严 |
| `main.py` 无 HTML | 未安装 Allure CLI，或 `allure generate` 失败（查看终端输出） |
